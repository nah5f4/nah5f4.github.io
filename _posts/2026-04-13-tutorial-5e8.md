# Industrial I/O (IIO) e o Driver Dummy

Nos tutoriais 7 e 8, o foco foi o subsistema IIO (Industrial I/O) do Kernel Linux. O objetivo foi entender como o kernel gerencia sensores e como podemos simular esses dispositivos.

## 1. Anatomia do iio_simple_dummy

O subsistema IIO utiliza canais para representar dados (como eixos X, Y, Z). Estudei a estrutura básica do driver dummy:

* struct iio_chan_spec: Define as características do canal (tipo, precisão, e formato do dado).
* read_raw / write_raw: Funções que permitem ler dados ou configurar o sensor via sysfs.
* struct iio_dev: Representa o dispositivo IIO completo para o sistema.

---

## 2. Habilitação e Compilação

Ativei o driver dummy via nconfig (em `Device Drivers -> Industrial I/O support`) e compilei o módulo para a arquitetura ARM64:

$ make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- M=drivers/iio/dummy

---

## 3. Experimentos Práticos (Tutorial 8)

Com o módulo compilado, realizei os seguintes passos na VM:

### Carregamento do Módulo
Para carregar o driver, precisei transferir os arquivos .ko e utilizar o comando insmod. Devido à falta de suporte nativo no kernel da VM, também foi necessário compilar e carregar dependências como `industrialio-sw-device.ko` e `kfifo_buf.ko`.

### Criação do Dispositivo via ConfigFS
O driver dummy exige a criação manual do dispositivo através do sistema de arquivos de configuração:

# Montagem do configfs
$ sudo mkdir -p /mnt/iio_experiments
$ sudo mount -t configfs none /mnt/iio_experiments

# Criando uma instância do sensor
$ sudo mkdir -p /mnt/iio_experiments/iio/devices/dummy/meu_device_teste

### Inspeção no Sysfs
Após a criação, o dispositivo apareceu em `/sys/bus/iio/devices/iio:device0`. Verifiquei os atributos gerados:

$ cat /sys/bus/iio/devices/iio:device0/name
meu_device_teste
$ cat /sys/bus/iio/devices/iio:device0/in_voltage0_raw
(exibe o valor simulado)
